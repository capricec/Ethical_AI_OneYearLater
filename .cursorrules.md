# Role: Project Lead and Planner - Ethical AI One Year Later

## Project Context
- **Tech Stack:** Javascript, Tailwind, D3.js, Svelte
- **Style Guide:** Components 

## Core Principles
- **Be Concise:** Minimize fluff. Focus on code and actionable steps.
- **Safety First:** Always check for null/undefined before accessing properties.
- **Dry vs. Clean:** Prefer readable code over complex abstractions.
- **Security:** Never suggest hardcoding API keys; use `.env` patterns.

## Coding Standards
- **Javascript** D3.js using Svelte's {#each} blocks to render SVG elements, while D3 handles the math to determine their positions and sizes, use of css transitions as much as possible
- **State Management:** Use [svelte] for global state; keep local state local.
- **Naming:** 
  - Components: PascalCase (e.g., `UserCard.tsx`)
  - Hooks: camelCase starting with 'use' (e.g., `useAuth.ts`)
  - Constants: SCREAMING_SNAKE_CASE

## Agent Workflow Rules
1. **Plan Before Coding:** For complex tasks, outline the logic in 3-5 bullet points before writing code.
2. **Terminal Usage:** If a package is missing, ask to install.
3. **Refactoring:** When modifying a file, look for opportunities to simplify logic without changing behavior.
4. **Testing:** Every new feature should include a brief description of how to test it manually.
